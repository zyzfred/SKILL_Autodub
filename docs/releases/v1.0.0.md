# Release Notes: v1.0.0

## English

`v1.0.0` is the first sealed public release of the `video-target-subtitles` Codex skill.

### What This Release Ships

- A deployable Codex skill folder: `video-target-subtitles/`
- Bundled Python scripts for probing media, extracting audio, ASR, translation, timing polish, subtitle export, and linting
- Skill metadata for Codex UI via `agents/openai.yaml`
- Reference docs for runtime configuration and subtitle localization rules

### Default Behavior in v1.0.0

- Preserves short emphasis cues by default
- Splits long source cues but does not merge short cues just because they are short
- Uses rule-based semantic repair only for semantically incomplete fragments
- Exports subtitles with minimal punctuation by default
- Wraps long subtitle cues across semantic lines instead of forcing extra cue splits by default
- Treats `max-lines` as a warning-only review rule, not a hard export failure

### Deployment

Clone the repository and copy or symlink `video-target-subtitles/` into your Codex skills directory:

```bash
git clone https://github.com/zyzfred/SKILL_Autodub.git
mkdir -p "$CODEX_HOME/skills"
cp -R SKILL_Autodub/video-target-subtitles "$CODEX_HOME/skills/video-target-subtitles"
```

Expected deployed path:

```text
$CODEX_HOME/skills/video-target-subtitles
```

### Runtime Requirements

- `ffmpeg`
- `ffprobe`
- Python 3
- `uv` recommended
- Python packages: `dashscope`, `openai`

Install Python dependencies:

```bash
uv pip install dashscope openai
```

### Verified Runtime Snapshot

- ASR model: `fun-asr-realtime-2026-02-28`
- Translation model: `qwen3.5-flash-2026-02-23`
- ASR provider: DashScope FunASR realtime recognition
- Translation provider: DashScope OpenAI-compatible endpoint

### Validation Result

Validated against sample input:

```text
/Users/mac/Desktop/AI-Workspace/autodub/data/真人.mp4
```

Verified outputs:

```text
/Users/mac/Desktop/AI-Workspace/autodub/output/真人-v053.en.srt
/Users/mac/Desktop/AI-Workspace/autodub/output/真人-v053.en.vtt
```

Lint summary:

- `54` cues
- `0` errors
- `1` warning

Accepted residual warning:

- `Cue 14: 3 lines exceeds max 2`

This warning is accepted in `v1.0.0` because the resulting wrap is semantically clean and consistent with the release rule that `max-lines` is review-only.

### Known Limits

- FunASR sentence boundaries are not publication-ready on their own
- Some ASR text can still be wrong, not just badly segmented
- Model-based semantic repair exists in code but is not the default stable path
- One multi-sentence cue in the validated sample still wraps to 3 lines

### Next Likely Improvements

- Improve semantic wrapping for multi-sentence cues
- Add optional cue splitting only for genuinely unwrappable dense cues
- Add glossary or proper-name protection for translation stability

## 简体中文

`v1.0.0` 是 `video-target-subtitles` Codex skill 的第一个正式封版公开版本。

### 本次发布内容

- 可直接部署的 Codex skill 目录：`video-target-subtitles/`
- 完整的 Python 脚本：媒体探测、音频提取、ASR、翻译、时间轴微调、字幕导出和 lint
- Codex UI 元数据：`agents/openai.yaml`
- 运行配置与字幕本地化规则参考文档

### v1.0.0 默认行为

- 默认保留短重点句
- 长 source 句会切分，但不会因为“短”就自动合并
- 规则版 semantic repair 只处理语义残缺片段
- 导出字幕默认使用最小标点
- 长字幕默认按语义换行，不强制拆成更多 cue
- `max-lines` 仅作为 warning，不是导出失败条件

### 部署方式

克隆仓库后，将 `video-target-subtitles/` 复制或软链接到 Codex skills 目录：

```bash
git clone https://github.com/zyzfred/SKILL_Autodub.git
mkdir -p "$CODEX_HOME/skills"
cp -R SKILL_Autodub/video-target-subtitles "$CODEX_HOME/skills/video-target-subtitles"
```

部署后的期望路径：

```text
$CODEX_HOME/skills/video-target-subtitles
```

### 运行依赖

### 当前限制

- 模型版 semantic repair 已存在，但不是当前默认稳定路径
- 当前验证样例里仍有一个多句 cue 会换成 3 行

### 下一步可能迭代

- 继续优化多句 cue 的语义换行
- 只在真正无法合理换行时再启用 cue 拆分
- 增加 glossary / 专有名词保护来提升翻译稳定性
