# Release Notes: v1.1.1

## English

`v1.1.1` is the current repository release tag for the subtitle workflow baseline after adding OCR fallback to source-text extraction.

### What This Repository Release Includes

- syncs `video-target-subtitles` to `v1.1.1`
- adds automatic OCR fallback when FunASR fails or returns no usable source segments
- records whether source extraction succeeded through speech ASR or OCR fallback in run summaries
- updates bilingual skill docs and runtime configuration to document `SUBTITLE_OCR_MODEL`

### Repository Workflow Baseline

1. `video-target-subtitles` generates multilingual subtitle assets
2. the source-text stage prefers speech ASR and falls back to OCR when needed
3. human QA approves subtitle text and timing
4. `video-subtitle-delivery` packages reviewed subtitles for final delivery
5. a separate dubbing workflow may consume the approved assets downstream

### Validation Snapshot

- `video-target-subtitles/scripts/*.py` compile successfully
- `video-subtitle-delivery/scripts/*.py` compile successfully
- the new fallback entrypoints parse their CLI arguments successfully

## 简体中文

`v1.1.1` 是当前仓库级发布标签，对应“source-text 提取新增 OCR 兜底”后的字幕工作流基线。

### 这次仓库发布包含

- 将 `video-target-subtitles` 同步到 `v1.1.1`
- 当 FunASR 失败或没有产出可用 source segments 时，新增自动 OCR 兜底
- 在 run summary 中记录 source extraction 最终来自语音 ASR 还是 OCR
- 更新中英双语 skill 文档与运行时配置，补充 `SUBTITLE_OCR_MODEL`

### 仓库工作流基线

1. `video-target-subtitles` 负责生成多语种字幕资产
2. source-text 阶段优先使用语音 ASR，必要时回退到 OCR
3. 人工 QA 审核字幕文本与时间轴
4. `video-subtitle-delivery` 负责把审核后的字幕打包成最终交付物
5. 独立的配音工作流可以在下游消费这些确认后的资产

### 验证快照

- `video-target-subtitles/scripts/*.py` 编译通过
- `video-subtitle-delivery/scripts/*.py` 编译通过
- 新增的 fallback 脚本入口可以正常解析命令行参数
