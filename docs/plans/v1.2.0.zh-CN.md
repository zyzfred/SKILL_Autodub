# v1.2.0 Plan

[English](v1.2.0.md) | [简体中文](v1.2.0.zh-CN.md)

## 目标

下一轮仓库迭代聚焦两条前进路径：

- 更强的多语种字幕批量生成
- 一个边界清晰、独立存在的多语种配音 skill 契约

## 为什么要有这份计划

仓库现在已经把“字幕生产”和“字幕交付”拆成稳定的两段。下一步应该在不打穿边界的前提下扩能力，而不是再把几个阶段揉回一起。

## 主要目标

1. 把多语种字幕生成扩展到“一次运行支持多个目标语言”
2. 为配音建立独立 skill 契约，而不是继续挤占字幕或交付 skill
3. 保持字幕到 delivery 的交接稳定且机器可读

## 拟议工作项

1. 为 `video-target-subtitles` 增加 multi-target batch 模式
2. 输出按语言拆分的 batch summary，以及更清晰的 rerun 元数据
3. 新增 `video-target-dubbing/` skill 骨架，并补全中英双语文档
4. 明确配音输入契约：
   已审核字幕、时间轴 JSON、speaker map、voice config，以及输出音频布局
5. 写清楚非目标，避免配音逻辑重新泄漏回字幕生成或 delivery 打包

## 验收标准

- 一次字幕运行可以稳定地产出多个目标语言
- rerun 能只重试失败语言，而不用重跑所有已成功输出
- 新配音 skill 的契约文档具备中英文版本
- 仓库仍然保持字幕生产、字幕交付和配音三个独立阶段

## 非目标

- 把配音合并进 `video-target-subtitles`
- 把配音合并进 `video-subtitle-delivery`
- 用自动配音默认值掩盖 QA 阶段
- 除非配音元数据真的需要，否则不重做当前 delivery 包结构
