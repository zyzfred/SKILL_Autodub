# Archived Plan: v1.0.1

[English](v1.0.1.md) | [简体中文](v1.0.1.zh-CN.md)

这份归档计划仅保留作历史参考。仓库目前已经同步到 `video-target-subtitles v1.1.0`。

跟踪分支：`codex/v1.0.1`

## 目标

发布一个范围较窄的后续版本，在不打扰当前 `v1.0.0` 行为基线的前提下改善语义换行。

## 当时的基线

`v1.0.0` 当时保证：

- 默认保留短重点句
- semantic repair 只合并残缺片段
- 导出字幕默认使用最小标点
- 长句默认按语义换行，而不是拆成更多 cue
- 已验证样例为 `0` 个错误，`1` 个已接受 warning

## 当时的主要目标

尽量降低或消除已验证样例中剩余的 warning：

- `Cue 14: 3 lines exceeds max 2`

## 约束

- 保持 target 侧拆 cue 为 opt-in
- 默认继续保持弱标点最小化
- 不回退短重点句保留行为
- 不引入新的时间轴重叠或空 cue

## 当时拟议工作项

1. 优化 `segments_to_subtitles.py` 中的语义换行启发式
2. 在空间允许时优先句级换行点，而不是从句级换行点
3. 使用 `v1.0.0` 的既有样例视频重新验证
4. 只有默认导出行为变化时才更新 release notes

## 验收标准

- 已验证样例依旧保持 `0` 个 lint error
- 剩余 warning 数量保持在 `1` 或更好
- 如果 warning 数量改善，明确记录具体变化
- README 中描述的部署步骤不发生回退

## 非目标

- 替换 ASR 后端
- 修改默认翻译提供方
- 默认启用模型版 semantic repair
- 把 wrapped-line warnings 变成硬失败
