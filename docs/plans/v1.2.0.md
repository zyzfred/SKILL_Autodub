# v1.2.0 Plan

[English](v1.2.0.md) | [简体中文](v1.2.0.zh-CN.md)

## Goal

Ship the next repository iteration around two forward paths:

- stronger multilingual subtitle generation at scale
- a cleanly separated multilingual dubbing skill contract

## Why This Plan Exists

The repository now has a stable split between subtitle production and subtitle delivery. The next step should extend capability without collapsing those boundaries again.

## Primary Targets

1. Extend multilingual subtitle generation beyond one target language per run
2. Define a dedicated dubbing-skill contract instead of overloading subtitle or delivery skills
3. Keep the subtitle-to-delivery handoff stable and machine-readable

## Proposed Work Items

1. Add a multi-target batch mode to `video-target-subtitles`
2. Emit per-language batch summaries and clearer rerun metadata
3. Introduce a new `video-target-dubbing/` skill skeleton with bilingual docs
4. Define the dubbing input contract:
   approved subtitles, timing JSON, speaker map, voice config, and output audio layout
5. Document strict non-goals so dubbing does not leak back into subtitle generation or delivery packaging

## Acceptance Criteria

- one subtitle run can target multiple output languages deterministically
- reruns can isolate failed languages without rerunning all successful outputs
- the new dubbing skill contract is documented in both English and Simplified Chinese
- the repository still keeps subtitle production, delivery, and dubbing as separate stages

## Non-Goals

- merging dubbing into `video-target-subtitles`
- merging dubbing into `video-subtitle-delivery`
- hiding QA behind automated dubbing defaults
- reworking the current delivery package layout unless dubbing metadata truly requires it
