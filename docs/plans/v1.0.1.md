# v1.0.1 Plan

Tracking branch: `codex/v1.0.1`

## Goal

Ship a narrow follow-up release that improves semantic wrapping without disturbing the current `v1.0.0` behavior baseline.

## Current Baseline

`v1.0.0` currently guarantees:

- short emphasis cues are preserved by default
- semantic repair only merges incomplete fragments
- exported subtitles default to minimal punctuation
- long cues are wrapped semantically instead of being split into extra cues by default
- the validated sample has `0` errors and `1` accepted warning

## Primary Target

Reduce or eliminate the remaining accepted warning in the validated sample:

- `Cue 14: 3 lines exceeds max 2`

## Constraints

- Keep target-side cue splitting opt-in
- Keep weak punctuation minimal by default
- Do not regress short emphasis cue preservation
- Do not introduce new timing overlaps or empty cues

## Proposed Work Items

1. Improve semantic wrapping heuristics in `segments_to_subtitles.py`
2. Prefer sentence-level wrap points before clause-level wrap points when space allows
3. Revalidate against the existing sample video used for `v1.0.0`
4. Update release notes only if the default exported behavior changes

## Acceptance Criteria

- Validated sample still has `0` lint errors
- The remaining warning count stays at `1` or improves
- If warning count improves, document the exact delta
- No regression in README-described deploy steps

## Non-Goals

- replacing the ASR backend
- changing the default translation provider
- enabling model-based semantic repair by default
- turning wrapped-line warnings into hard failures
